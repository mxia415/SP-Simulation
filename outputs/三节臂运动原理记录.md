## 坐标与单位

- 单位：毫米。
- 当前机构主平面为 `Y=0` 的 XZ 平面；侧向镜像使用 Y 偏置。
- Three.js 世界坐标使用 X/Y/Z：
  - X：水平轴向。
  - Y：侧向/宽度方向。
  - Z：高度。
- 默认/标定姿态：`90 / 90 / 90 / 0 / 180`。
- 底座-臂1主轴、臂段、连杆、电缸和 Tool 的球棍模型是运动真源；GLB 只作为外观层跟随球棍点位。

## 电缸与连杆逻辑

- 电缸杆是变长构件；其长度由球棍两端实时距离决定。
- 连杆A、B是固定长度刚性杆；连杆两端都以球铰/球体显示。
- 连杆共同端点由固定长度连杆求交确定，电缸只跟随该共同端点，不参与固定长度求交。
- 所有电缸和连杆 GLB 后续都使用同一策略：参考姿态记录 GLB 姿态，运行时按球棍端点做双端锚定，并使用稳定侧向轴避免长轴自转。

## GLB 模型策略

当前 GLB 参考姿态统一为：

`X=-3050, Y=-135, Z=-1590, Rx=0, Ry=0, Rz=-90, scale=1`

已导入模型：

| 分组 | 文件 |
|---|---|
| Base | `base.glb` |
| Base Link | `base_link.glb` |
| Arm | `arm1.glb`, `arm2.glb`, `arm3.glb` |
| Tool | `tool.glb` |
| 电缸1 | `cyl1_xn_base.glb`, `cyl1_xn_end.glb`, `cyl1_xp_base.glb`, `cyl1_xp_end.glb` |
| 电缸2 | `cyl2_xn_base.glb`, `cyl2_xn_end.glb`, `cyl2_xp_base.glb`, `cyl2_xp_end.glb` |
| 电缸3 | `cyl3_mid_base.glb`, `cyl3_mid_end.glb` |
| 连杆A | `link_a1_xn.glb`, `link_a1_xp.glb`, `link_a2_mid.glb` |
| 连杆B | `link_b1_xn.glb`, `link_b1_xp.glb`, `link_b2_mid.glb` |

模型策略：

- 运动 GLB 的位置由参考姿态本地锚点反算，不通过移动球棍点适配。
- Arm1/Arm2/Arm3/Tool 按各自臂段随动。
- 电缸和连杆按两端球棍点随动；禁止出现与球棍长轴不平行的自转。
- 右侧模型姿态面板默认折叠，已锁定的模型不再作为常规调参入口。

## 线性驱动与拖拽

- 拖拽对象是 Tool 末端的蓝色控制球，不是直接拖 GLB。
- 控制球位置与偏移后的 Tool 球棍末端重合。
- 鼠标按下后创建面向当前相机的拖拽平面，拖动目标通过逆解更新臂1/臂2/臂3/打印头/旋转。
- 拖拽时临时关闭 OrbitControls，释放后恢复。
- 线性路径保存起点时必须保存当前姿态；模拟时先恢复起点姿态，再沿世界目标插值求解。

## 渲染与视角

- 页面名称统一为 `SP-S模拟`，可视标识统一使用 `GL-3DPRT-SP-S`。
- 当前浅色 GLB 实体为非透明浅灰白、高粗糙、无金属、无自发光，并带低透明边线。
- 球棍模型不受 GLB 浅色/透明材质策略影响。
- 相机保持原稳定参数：`PerspectiveCamera(42, 1, 0.08, 85)`；不要改成 `near=10`，否则放大视角时模型会消失。
